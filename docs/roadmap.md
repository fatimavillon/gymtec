# Roadmap de GYMTEC

## Fase 1: Ordenar el proyecto en GitHub

Estado: Completado.

Objetivo:
Dejar el repositorio organizado para que el equipo pueda trabajar de forma ordenada.

Responsable principal:
Mish.

Entregables:
- Estructura base del repositorio.
- README inicial.
- Carpetas para datos, notebooks, src, app, docs, reports y tests.
- Gitignore configurado.

---

## Fase 2: Definir el pipeline de datos

Objetivo:
Definir cómo los datos entran, se limpian, se transforman y se preparan para el modelo.

Responsable principal:
Mish con apoyo de Bihonda.

Entregables:
- Documento del pipeline.
- Diccionario de datos.
- Scripts iniciales de carga y limpieza.
- Dataset procesado base.

Tareas:
- Definir fuentes de datos.
- Documentar columnas esperadas.
- Crear reglas de limpieza.
- Crear estructura de feature engineering.
- Validar qué datos necesita el modelo.

---

## Fase 3: Modelo predictivo de aforo

Objetivo:
Entrenar y evaluar un modelo que prediga el aforo del gimnasio por día y horario.

Responsable principal:
Bihonda.

Entregables:
- Notebook de modelamiento.
- Métricas del modelo.
- Feature importance.
- Modelo seleccionado.
- Predicciones exportables.

---

## Fase 4: Recomendación personalizada de horarios

Objetivo:
Construir una lógica que recomiende horarios convenientes para asistir al gimnasio.

Responsable principal:
Mish con apoyo de Bihonda.

Entregables:
- Reglas de recomendación.
- Score por horario.
- Ranking de horarios recomendados.
- Explicación de la recomendación.

---

## Fase 5: UX/UI y visualización

Objetivo:
Diseñar las pantallas principales para estudiantes y administradores.

Responsable principal:
Fátima.

Entregables:
- Wireframes.
- Flujo de usuario.
- Diseño visual.
- Pantallas del dashboard/app.

---

## Fase 6: Integración del prototipo

Objetivo:
Integrar datos, modelo, recomendador y visualización en un prototipo funcional.

Responsable principal:
Mish.

Entregables:
- Prototipo integrado.
- Dashboard o app funcional.
- Documentación de uso.
- Presentación final.

---

## Fase 7: Validación y presentación final

Objetivo:
Validar que el producto sea entendible, útil y coherente con el problema.

Responsable principal:
Todo el equipo.

Entregables:
- Demo final.
- Slides.
- Reporte o memoria del producto.
- Evidencia del funcionamiento.
